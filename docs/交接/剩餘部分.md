# 这轮已经完成的事情

## 1. `0.2.1` 发布已经完成，`0.3.0` 已进入发版准备

- npm 上最新已发布版本是 `@statekit-vue/shared@0.2.1` 和 `@statekit-vue/vue@0.2.1`。
- 本地 workspace 已切到 `0.3.0` release-prep 版本线。

当前推荐的公开入口是：

- `EmptyState`
- `OnboardingState`
- `LoadingState`
- `ErrorState`
- `PermissionState`
- `UpgradeState`
- `SuccessState`

底层现在保留 21 个 preset recipes。旧的场景名导出依然存在，但已经作为 deprecated compatibility exports 处理。

## 2. onboarding category pilot 已落地（未发版）

- 这轮已经把 onboarding 从 empty 中拆出，作为独立 category 对外提供 `OnboardingState`。
- shared 元数据新增并补齐了 `onboarding-workspace`、`onboarding-members`、`onboarding-integration`，用于承载首次创建工作区、邀请成员、连接初始集成这三个 onboarding 节点。
- `first-project` 没有删除，但现在更适合表达"工作区已经存在，只是还没有第一个项目"的 empty-state 过渡语义。
- docs 首页、recipes 列表、recipe detail、docs 示例页和 `examples/vite-vue-admin` 都已经同步到 7 类 / 21 recipes 的新口径。

## 3. OnboardingActivationView 新增（未发版）

- 新增 `apps/docs/src/views/examples/OnboardingActivationView.vue`，展示完整 first-run 激活链路：创建 workspace → 邀请成员 → 连接集成 → 完成。
- 新增 `/examples/onboarding-activation` 路由，`/examples` 默认跳转至此。
- 这是 docs 里第一个端到端串联多个 category 的流程型示例页，使"onboarding 不是 generic empty state"这条语义差异有了可视化产品上下文。

## 4. Playwright 测试覆盖全面扩充（未发版）
